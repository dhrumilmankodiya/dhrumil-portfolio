## 1. Concept & Vision

A world-class personal portfolio for Dhrumil Mankodiya — 23-year-old Product Designer & AI Product Manager building an empire. The site should feel like a **premium product in itself**: confident, minimal, fast, and memorable. It tells the story of a builder who ships, not just a designer who designs.

**Tone:** Confident, direct, ambitious. Zero fluff. Every word earns its place.
**Feel:** Premium startup founder meets elite product designer. Think Linear.app meets Brittany Chiang meets Pablo Stanley.
**Animation:** Purposeful — entrance animations feel alive but never slow. Scroll-driven reveals. Micro-interactions on hover.

## 4. Design System

### 4.1 Colors
```
Primary:        #000000 (Black — dominant)
Accent:         #D4AF37 (Gold — accent, sparingly)
Accent Light:   #F5E6B8 (Light gold — badges, highlights)
Background:     #FFFFFF (White)
Surface:        #F8F8F8 (Off-white — section alternation)
Text Primary:   #000000
Text Secondary: #6B7280 (Gray 500)
Text Muted:     #9CA3AF (Gray 400)
Border:         #E5E7EB (Gray 200)
Dark BG:        #111111 (for dark mode or contrast sections)
```

### 4.2 Typography
```
Font Family:    Inter (Google Fonts) — clean, modern, highly readable
Heading:        Inter 700 / 800 (Bold/ExtraBold)
Body:           Inter 400 / 500 (Regular/Medium)
Mono:           JetBrains Mono (for code snippets if any)

Scale:
- Hero name:     80px / 96px desktop, 48px mobile  |  weight 800
- H1:            48px desktop, 36px mobile          |  weight 700
- H2:            36px desktop, 28px mobile          |  weight 700  
- H3:            24px desktop, 20px mobile          |  weight 600
- Body:          18px / 16px                        |  weight 400
- Small/Label:   14px / 12px                        |  weight 500
- Tag/Badge:     12px                               |  weight 500, uppercase, tracked

Line heights:    1.2 for headings, 1.6 for body
Letter spacing:  0.05em for labels/uppercase
```

### 4.3 Spacing
```
Base unit:      4px
Scale:          4, 8, 12, 16, 24, 32, 48, 64, 96, 128px
Section padding: 96px vertical desktop, 64px mobile
Card padding:    24px
Gap:             24px default, 16px compact
```

### 4.4 Components

**Buttons:**
- Primary: Black bg, white text, rounded-full, px-8 py-3, hover: gray-800
- Secondary: White bg, black border, rounded-full, hover: gray-50 bg
- Text link: Underline on hover, no button styling

**Cards:**
- White bg, rounded-2xl, subtle shadow, hover: lift (translateY -2px + shadow increase)
- Border: 1px gray-200 or no border depending on section

**Badges/Pills:**
- Gray-100 bg, rounded-full, px-4 py-2, text-sm
- Dark variant: black bg, white text

**Nav:**
- Desktop: horizontal links, space-x-8
- Mobile: hamburger → full-screen overlay
- Transparent on hero → white on scroll (sticky)

**Images:**
- Aspect ratios: 16:9 for hero, 4:3 for project cards
- Lazy loading, blur-up placeholder
- Rounded corners: rounded-xl for cards, rounded-2xl for larger images

## 5. Animation Specifications

### 5.1 Entrance Animations
```
Hero name:      Stagger each character — opacity 0→1, translateY 20→0, 600ms ease-out, 30ms stagger
Sections:       Fade in + translateY 40→0, 500ms ease-out, triggered on scroll-into-view
Cards:          Stagger 100ms between items, same fade+translate
Stats numbers:  Count-up animation when in viewport
```

### 5.2 Micro-interactions
```
Buttons:        Scale 1→0.98 on press, 100ms
Cards:          translateY 0→-4px + shadow increase on hover, 200ms ease
Links:          Underline grows from left on hover
Badges:         Slight scale 1→1.02 on hover
Scroll:         Smooth scroll behavior
```

### 5.3 Scroll Animations
```
Section reveals: Intersection Observer — trigger at 20% visibility
Nav:             Background fade in after 100px scroll
Progress:        Optional: scroll progress indicator at top
```

## 9. Key Decisions

1. **Single page, scroll-based** — no routing, no complexity
2. **SSG only** — no server, no API routes, pure static
3. **Content in code** — no CMS, no database. Content in `lib/data.ts`
4. **Existing images** — reuse current website images
5. **Lucide icons** — consistent stroke, always 24px unless specified
6. **No dark mode for v1** — keep it simple, one theme
7. **No blog/newsletter** — keep focused on portfolio
